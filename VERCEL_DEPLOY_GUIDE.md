## 📋 ウェブフリコム振込データ詳細仕様

### ファイルフォーマット

| 行番号 | 内容        | 長さ    |
|--------|-------------|---------|
| 1      | ヘッダー行  | 120文字 |
| 2〜N   | データ行    | 120文字 |
| N+1    | トレーラ行  | 120文字 |
| N+2    | エンド行    | 120文字 |

### ヘッダー行（1行目）
```
委託者コード (14桁) + 空白 (90桁) + 振込指定日 (8桁) + 空白 (8桁)
例: 12100000000000 + (90桁空白) + 03250000 + (8桁空白)
```

### データ行（2〜N行目）
```
銀行コード (5桁)          例: 29900 (ゆうちょ銀行)
+ 空白 (15桁)
+ 支店コード (3桁)        例: 328
+ 空白 (19桁)
+ 会員コード (8桁)        例: 12345678
+ 受取人名 (カナ30桁)     例: ヤマダタロウ
+ 空白 (6桁)
+ 金額 (10桁)             例: 0000009428 (9,428円)
+ 新規コード (1桁)        例: 1
+ ゼロ (12桁)             例: 000000000000
+ 会員コード下7桁 (7桁)   例: 2345678
+ ダミーコード (1桁)      例: 0
+ 新規コード (1桁)        例: 1
+ ゼロ+空白 (16桁)        例: 0 + (15桁空白)
```

### トレーラ行（最終行-1）
```
"8" + 件数合計 (6桁) + 金額合計 (12桁) + 空白 (101桁)
例: 8000038000001516621 + (101桁空白)
    ↑       ↑
    38件   1,516,621円
```

### エンド行（最終行）
```
"9" + 空白 (119桁)
```

## ⚠️ トラブルシューティング

### **Vercelビルドエラーが発生した場合**

1. **Vercelダッシュボードでログ確認**
   - https://vercel.com/dashboard
   - プロジェクト → Deployments → エラーのあるデプロイをクリック
   - ビルドログを確認

2. **よくあるエラーと解決方法**

   **エラー: `DATABASE_URL environment variable not set`**
   ```
   解決策: Vercel Settings → Environment Variables で DATABASE_URL を設定
   ```

   **エラー: `Prisma Client initialization error`**
   ```
   解決策: ビルドコマンドに prisma generate が含まれているか確認
   package.json の build スクリプト: "prisma generate && next build"
   ```

   **エラー: `Module not found: Can't resolve '@/lib/prisma'`**
   ```
   解決策: tsconfig.json の paths 設定を確認
   ```

### **データが表示されない場合**

1. **ボーナス計算が確定されているか確認**
   - https://www.viola-pure.xyz/admin/bonus-run にアクセス
   - 対象月のボーナス計算が「**確定済み**」になっているか確認
   - 「下書き」の場合は確定処理を実行

2. **口座情報が登録されているか確認**
   - https://www.viola-pure.xyz/admin/mlm-members にアクセス
   - 会員の口座情報（口座番号、口座名義）が登録されているか確認
   - 口座情報がない会員はウェブフリコムデータに含まれません

3. **環境変数を確認**
   - Vercel Settings → Environment Variables
   - `DATABASE_URL`が正しく設定されているか確認
   - 変更した場合は再デプロイが必要

### **ダウンロードしたファイルが文字化けする場合**

**ウェブフリコムデータ（.txt）**:
- **Shift_JIS対応エディタ**で開いてください
- 推奨エディタ:
  - Windows: メモ帳、サクラエディタ、秀丸エディタ
  - Mac: CotEditor、mi
- VSCodeで開く場合:
  1. ファイルを開く
  2. 右下の文字コードをクリック
  3. 「Reopen with Encoding」→「Shift JIS」を選択

**CSVファイル**:
- Excel、Googleスプレッドシートで開く
- BOM付きUTF-8で出力されているため、自動認識されます

### **APIエラーが発生する場合**

**エラー: `404 Not Found`**
```
原因: ボーナス計算結果が存在しない
解決策: /admin/bonus-run で対象月のボーナス計算を実行・確定
```

**エラー: `500 Internal Server Error`**
```
原因: データベース接続エラー、またはデータ不整合
解決策:
1. Vercel Function Logs を確認（Vercel Dashboard → Logs）
2. DATABASE_URL が正しく設定されているか確認
3. データベース接続が正常か確認（Neon Dashboard で確認）
```

## 🔧 環境変数設定（Vercel）

既に設定されているはずですが、念のため確認：

### 必須環境変数

| 変数名 | 値 | 説明 |
|--------|-----|------|
| `DATABASE_URL` | `postgresql://neondb_owner:...` | Neon PostgreSQL接続URL |
| `NEXTAUTH_SECRET` | ランダム文字列 | Next-Auth認証シークレット |
| `NEXTAUTH_URL` | `https://www.viola-pure.xyz` | 本番URL |

### 設定方法

1. https://vercel.com/dashboard にアクセス
2. プロジェクトを選択 → **Settings** タブ
3. 左サイドバー → **Environment Variables**
4. 上記の変数が設定されているか確認
5. 変更した場合は **Redeploy** が必要
